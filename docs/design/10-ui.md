# 10 — UI

Last updated: 2026-04-22

The UI is server-rendered HTML (Askama) plus a small amount of vanilla JS for interactivity.
No build step, no framework.

## Pages

### Home — `/`

Two sections:

- **Directories** — directory collections with `hidden = 0`.
- **My Collections** — custom collections, with a "+ New Collection" card.

Each card shows the collection name, video count, and a 2×2 mosaic of recent thumbnails.

### Collection page — `/collections/:id`

- Header: name, count, **🎲 Random** button (hotkey `R`).
- For custom collections: Rename, Delete, Add Videos buttons.
- For directory collections: inline label edit only.
- Grid of video cards; each card has a hover-scrub preview powered by the cached
  preview sheet + VTT.

### Video detail page — `/videos/:id`

- Title: filename without extension. Full filename in the `title=` tooltip.
- Large poster with a scrub strip below it. The strip is informational only — it shows
  the time-at-cursor but clicking does not set a start time.
- Metadata: duration, resolution, codec, size, directory label, full path, last-watched,
  `Resume at HH:MM:SS` when applicable.
- Actions:
  - **Play**
  - **Resume** (only when `watch_history.position_secs > 0` and not completed)
  - **Re-roll** (only when `?cid=` query present)
  - **Add to collection…**
- Collection chips for all collections this video belongs to.
- Keyboard: `Space`/`Enter` = Play, `R` = Re-roll (when applicable), `Esc` = Back.

### History — `/history`

Chronological list with resume progress bars and a "clear" button per row.

### Settings — `/settings`

- Directories table: label, path, # videos, activity, actions (Rename, Rescan, Remove…).
- The **Remove…** button opens a modal with two options:
  - **Soft remove** (primary button): hide from the UI while preserving watch
    history, custom collection memberships, and cached thumbnails. Re-adding the
    same path later brings everything back; cache files are reused if the video
    files haven't changed.
  - **Permanently delete** (danger button, behind an extra `confirm()`): deletes
    all database state for the directory and its videos plus the cached
    thumbnails and previews from disk. Cannot be undone.
- **Add Directory** button opens the directory picker modal.
- Global **Rescan All** button.
- Config info: port, player command, cache directory.

## Directory picker modal

- Breadcrumb path bar.
- List of child directories of the current path. Unreadable entries are shown disabled.
- "Up" and "Home" buttons.
- Optional `Label` input; defaults to the path.
- Starts at `ui_state.last_browsed_path` if set and still exists; otherwise `$HOME`.

## Hover-scrub component

Shared JS module reused by grid cards and the detail-page poster:

1. On mouseenter, fetch `/previews/<id>.vtt?v=<v>` once (cached in memory).
2. On mousemove, map mouse `x` across the element's width to a fraction, then to a cue.
3. Show the frame by setting the preview's background image and `background-position` to
   the cue's `xywh`.
4. On mouseleave, fall back to the poster.

Graceful degradation: if `preview_ok = 0`, the poster stays.
